HLD study method — active reconstruction only
1
Read once. Use Alex Xu (Vol 1+2) or Arpit Bhayani videos. One pass, take rough notes.
2
Close everything. Open blank doc. Design the full system from scratch — capacity, API, data model, components, scaling.
3
Identify gaps. Go back to resource only for what you missed. Not the whole thing.
4
Repeat day 3 and day 7. Re-design from memory. Mark mastered only when you can do it cleanly twice with no reference.
5
For each system answer all tradeoff questions (see per-system checklist). If you can't — you haven't mastered it yet.
LLD — 10 systems to code
LLD coding rules
Write actual code, not pseudocode
Every LLD system must be coded in Java. Interfaces, classes, enums — the real thing. Diagrams alone will not pass interviews at strong companies.
SOLID before clever
Single responsibility per class. Open for extension. Strategy pattern over if-else chains. Interviewers test whether you write extensible code, not just working code.
Extensibility questions
After every design ask: "If I needed to add X, how much would change?" If the answer is "a lot" — your abstractions are wrong. Redesign.
No cohort hand-holding after month 2
From month 3 onward, design each system independently before looking at any reference. The cohort is scaffolding, not the destination.
