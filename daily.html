Phase-wise focus breakdown
Non-negotiable rules
DSA: 25-minute rule
Attempt every problem independently for 25 min before looking at anything. This is where pattern recognition builds. Skip this and you can only solve problems you've memorised.
HLD: active reconstruction only
Read once, then close everything and design from scratch. Watching videos repeatedly is not studying. You only own a system when you can design it cold.
Projects: build, don't plan
Weekend project time is for writing code, not planning what to build. If you spend 45 min deciding what to do next, the session is wasted. Know the next task before you sit down.
Weekly plan: 20 min max
Sunday evening. What 3 things must happen this week? Write them. Done. Any more planning time is procrastination with extra steps.
Mock interviews: from month 3
Minimum 2 per week from month 3. Pramp for DSA. Interviewing.io or peer for system design. Record every session. Debrief same day. No exceptions.
No context switching
Weekday sessions have one DSA block and one HLD/LLD block. Not three things in parallel. Deep work on fewer things beats shallow work on everything.
